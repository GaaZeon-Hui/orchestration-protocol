严谨

状态机全览

四重安全校验

① CAS 乐观锁

每次推进先读取 revision，UPDATE 带 WHERE revision=?。rowcount=0 即为并发冲突 → RuntimeError → 重试。纯 SQLite 原子操作，无需 Redis。

② SQL Trigger 兜底

BEFORE UPDATE 触发：revision 必须 +1，转移路径必须匹配硬编码白名单。即使绕过 Python 裸写 SQL 也无法破坏。

③ 权限矩阵

Worker 不能推 Orch stage，Reviewer 只能推自己 stage。ROLE_PERMISSIONS 在 Python 层拦截，PermissionError 直接拒绝。

④ 字段白名单

ALLOWED_COLUMNS 静默过滤非法列名。写出非白名单字段 → 不写入，不会报错也不会泄露。
